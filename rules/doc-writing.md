# 문서 작성 원칙 — terraform-practice

## 언어
- 문서(README, CLAUDE.md)는 **한국어**로 작성
- AWS 서비스명, CLI 명령어, Terraform 키워드는 영어 그대로
- 경어체보다 간결한 서술체: `~다.`, `~한다.`

## README 구조 (EKS README 스타일 기준)
1. **개요** — 모듈 목적과 프로비저닝 리소스
2. **사전 요구사항** — 선행 모듈, 권한, 도구 버전
3. **설정 값** — 주요 변수와 기본값 표
4. **배포 순서** — make 명령어 단계
5. **배포 확인** — AWS CLI 확인 명령
6. **삭제** — destroy 방법 + 주의사항
7. **트러블슈팅** — 자주 겪는 문제

## 코드 주석 패턴

```hcl
### =============================================================================
### modules/example/main.tf
### 리소스에 대한 설명
### =============================================================================

### ---------------------------------------------------------------
### 섹션 제목
### 부가 설명
### ---------------------------------------------------------------
resource "aws_example" "this" {
  # 인라인 주석은 한국어
}
```

## 표 형식
- 환경별 설정 차이는 반드시 표로 비교 (dev vs prod)
- 변수 목록은 `| 변수 | dev | prod | 설명 |` 형식

## 주의사항 표시
- 위험한 작업: `> **주의**:` 경고 블록 사용
- prod 전용 설정: `> **prod 전용**:` 명시
